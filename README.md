# Seed Companion App

Welcome to the **Seed Companion App** repository! This is the parent-facing mobile application designed as a companion to the **Seed Storybox**—an AI-driven, Arduino UNO Q-powered storytelling device for children aged 6-8.

This repository contains a high-fidelity, fully interactive web prototype of the app as it would appear on a parent's smartphone, along with a built-in **Hardware Simulator** that replicates the physical storytelling device.

## 🌟 Prototype Features

The prototype features a split-screen design on desktop:
- **Left (Mobile Mockup)**: Runs the parent companion app, styled inside a smartphone frame. It behaves exactly as it would on a phone (switches to fullscreen on actual mobile viewports).
- **Right (Storybox Simulator)**: Represents the top of the physical acrylic and wooden Storybox device, including the 4 pegs, NeoPixel LED status ring, and big physical buttons.

### 1. Parent Transparency (Story Archive)
- Access and review all past stories generated by the Storybox.
- Sort and search stories by date, character, or food group ratios.
- **Narrator Player**: Listen to the exact story read out loud to the child with a synchronized canvas-based audio wave animation. Uses the HTML5 Web Speech Synthesis API.
- **Dietary Insights**: View clear summaries explaining the nutrition-related sub-arcs of the story (e.g. *Why did Barnaby Bear climb a hill? Grains give slow-release energy!*).
- **Parent Reflection Notes**: Add, edit, and save notes to keep track of what characters or moral lessons the child reacted to.

### 2. Nutrition Analytics & Milestones
- **Weekly Balance Chart**: A dynamic donut chart illustrating the percentages of stacked food category beads Liam has used (Meat/Fish, Veggies/Fruits, Grains, Dairy).
- **Milestone Badges**: Gamified badges that unlock automatically based on the child's bead selections (e.g. *Veggie Explorer* when they complete 3 or more veggie-heavy stories).
- **Parent Coach Assessment**: Context-aware parenting advice based on the balance of stories. For example, it suggests how to encourage the child to stack green veggie beads if they choose grains too often.

### 3. Storybox Settings & Parental Controls
- **Max Volume Limiter**: Remotely restrict the maximum volume of the physical Storybox speaker.
- **Voice Profiler**: Switch TTS reading styles (Warm Female, Adventurous Male, Soft Bedtime) directly from the settings panel.
- **Bedtime Story Lock**: Prevent children from generating stories after a specific hour (e.g. 8:00 PM) to ensure healthy bedtime routines.
- **Wi-Fi Settings & Diagnostics**: Push Wi-Fi credentials to the simulated board and review raw analog sensor readouts (ADC) from the slot resistor ladders.

## 🛠️ Hardware Simulation Logic (Seed v2.2)

The Storybox simulator runs on the Seed v2.2 Storytelling rules, matching the constraints of the Arduino UNO Q board:
- **Resistor Ladders**: Stacking beads (max 5 per category) changes peg analog voltages. The simulator updates raw ADC values dynamically in the settings diagnostic panel.
- **Dominant Glow**: The NeoPixel LED ring color syncs with the dominant bead category stacked on the pegs, or cycles/mixes if they are balanced.
- **TTS Story Compiler**: Generates whimsical stories (260-380 words) using dynamic characters:
  - 🍖 Meat/Fish (Protein) -> *Barnaby Bear* (Animal Character)
  - 🥦 Veggies/Fruit (Vitamins) -> *Miss Broccoli* (Plant Character)
  - 🌾 Grains (Energy) -> *Gideon Grain* (Grain Character)
  - 🥛 Dairy (Calcium) -> *Captain Cheese* (Dairy Character)
- **Dairy = 0 Obstacle Patch**: If 0 dairy beads are stacked, obstacles in the story are limited to low-effort items (small twigs, puddles, pebbles) to reflect the lack of calcium strength.
- **5-Bead Protagonist Softening**: Stacking 5 beads of a single group triggers a softening patch—the protagonist moves with gentle determination (no strong physical verbs or timed precision actions).
- **Ensemble Movement Patch**: Avoids coordinated terms (like "perfect harmony" or "synchronized") to keep movements organic.
- **Bedtime Lock Enforcement**: If the bedtime lock is enabled in the parent settings, pressing "START STORY" on the simulator after hours will fail, logging a warning in the hardware console.

## 🚀 How to Run the Prototype

Since this is built with standard, highly optimized web standards (HTML5, CSS3, and ES6 Javascript), there are **no dependencies** to install. 

1. Clone this repository to your computer:
   ```bash
   git clone https://github.com/pariniim/seed_companion_app.git
   ```
2. Open `index.html` directly in any modern web browser (Chrome, Safari, Firefox, Edge).
3. Adjust the beads on the pegs in the right-hand panel, then press the green **START STORY** button!

*Note: All stories and settings are persisted in your browser's `localStorage` so they won't disappear when you refresh or reopen the page.*

## 🐍 Streamlit Hosting & Cloud Deployment

You can host and showcase this prototype online using Python and **Streamlit**.

### 1. Run Locally via Streamlit
If you have Python installed, you can run the Streamlit dashboard on your local machine:

1. Install Streamlit:
   ```bash
   pip install -r requirements.txt
   ```
2. Run the Streamlit application:
   ```bash
   streamlit run streamlit_app.py
   ```
3. Open the local address (typically `http://localhost:8501`) in your browser.

### 2. Deploy to Streamlit Community Cloud (Free Hosting)
To share a live URL of this prototype with your team members:

1. Push this repository (including `streamlit_app.py` and `requirements.txt`) to your GitHub account.
2. Visit [Streamlit Community Cloud](https://share.streamlit.io/) and log in with your GitHub account.
3. Click **New App**, select your repository and branch, and set the main file path to `streamlit_app.py`.
4. Click **Deploy!** Your app will be live at a custom shareable URL (e.g. `https://your-app.streamlit.app/`).
